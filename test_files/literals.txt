Total literal patterns: 28 (18 original + 10 escaped character tests)

Expected behavior:
- ASCII patterns: match when bytes are exactly as specified (with nocase variations)
- Wide patterns: match when text is encoded as UTF-16 LE (with null bytes between chars)
- XOR patterns: match when XORed with specified key range
- Base64 patterns: match when base64 encoded
- Fullword patterns: only match when surrounded by non-alphanumeric characters
- Private patterns: match found but not included in condition evaluation
- Escaped characters: 
  * \" matches literal double quote character
  * \\ matches literal backslash character
  * \r matches carriage return (0x0D)
  * \t matches horizontal tab (0x09)
  * \n matches newline/line feed (0x0A)
  * \xdd matches byte with hex value dd (e.g., \x41 = 'A')

Note: Wide string patterns with actual UTF-16 encoding are indicated by [WIDE_*] markers
followed by spaced characters. In a real binary file, these would be actual null-byte
separated characters (e.g., "wide" = 0x77 0x00 0x69 0x00 0x64 0x00 0x65 0x00).

For testing purposes, you may need to create a binary file with proper UTF-16 LE encoding
for wide string tests to work correctly. The spaced characters above are visual representations
only and won't match wide patterns in this ASCII text file.
